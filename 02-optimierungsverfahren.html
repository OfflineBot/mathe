Optimierungsverfahren

LP & Simplex · KKT · Newton · gradientenfrei · Metaheuristiken · Mehrziel

Moodle: OV-01 bis OV-07Übungsklausur inkl. Lösung

Übungen

Alle Klausur-Übungsaufgaben mit ausführlichen Lösungen findest du in der Begleitseite: Übungen — Optimierungsverfahren.

Überblick

Du suchst $x^* \in X$, das eine Zielfunktion $f$ minimiert oder maximiert — eventuell unter Nebenbedingungen. Die Verfahren unterscheiden sich nach: Linearität, Stetigkeit, Differenzierbarkeit, Konvexität, Zahl der Ziele, ob diskret oder kontinuierlich.

Was die Klausur prüft
  • LP grafisch in 2D + Simplex-Tableau für ein kleines LP.
  • KKT-Bedingungen in einem 2D-NLP mit einer Ungleichungs-NB.
  • Konvexitäts-Begriffe und Schluss "lokal = global".
  • Eine Metaheuristik beschreiben (Pseudocode, Stop-Kriterium, Parameter).
  • Pareto-Dominanz auf einer Tabelle und Skalarisierungs-Limitation bei nicht-konvexer Front.

1 · Grundlagen

1.1 Allgemeines Optimierungsproblem

Standardform $$ \min_{x \in X} f(x) \quad \text{u.d.N.} \quad g_i(x) \le 0,\; i = 1, \dots, m,\quad h_j(x) = 0,\; j = 1, \dots, p $$
  • $f: \mathbb{R}^n \to \mathbb{R}$ — Zielfunktion
  • $g_i$ — Ungleichungsnebenbedingungen
  • $h_j$ — Gleichungsnebenbedingungen
  • Zulässige Menge: $\mathcal{F} = \{x \in X : g_i(x) \le 0, h_j(x) = 0\}$.

Maximierung lässt sich immer auf Minimierung reduzieren: $\max f \equiv \min (-f)$ mit demselben Optimalpunkt $x^*$. Vorzeichen am Funktionswert nicht vergessen!

1.3 Konvexität

Definition

Eine Menge $C \subseteq \mathbb{R}^n$ ist konvex, wenn $\forall x, y \in C,\, \lambda \in [0,1]: \lambda x + (1-\lambda) y \in C$. Anschaulich: Verbindungsstrecke jedes Paares bleibt in $C$.

Eine Funktion $f: C \to \mathbb{R}$ auf konvexem $C$ ist konvex, wenn $$ f(\lambda x + (1-\lambda) y) \le \lambda f(x) + (1-\lambda) f(y). $$ Strikt konvex bei "$<$" für $x \ne y$, $\lambda \in (0,1)$.

Hauptsatz

Für ein konvexes Optimierungsproblem ($f$ konvex, $\mathcal{F}$ konvex) gilt: jedes lokale Minimum ist auch globales Minimum. Bei strikt konvexem $f$ ist das Minimum eindeutig.

Kriterien für Konvexität

  • $f \in C^2$: $f$ konvex ⟺ Hessematrix $\nabla^2 f(x) \succeq 0$ (positiv semidefinit) für alle $x$. Strikt konvex bei $\succ 0$.
  • Summe konvexer Funktionen ist konvex.
  • Max endlich vieler konvexer Funktionen ist konvex.
  • $f$ konvex, $g$ affin → $f \circ g$ konvex.
Beispiel

$f(x, y) = x^2 + 4 y^2$. Hesse $= \begin{pmatrix} 2 & 0 \\ 0 & 8 \end{pmatrix}$ positiv definit → strikt konvex.

$f(x) = x^3$ auf $\mathbb{R}$: $f''(x) = 6x$, kein einheitliches Vorzeichen → nicht konvex global. Auf $[0, \infty)$ konvex.

1.4 Optimalitätsbedingungen (unbeschränkt)

Notwendig 1. Ordnung

Bei differenzierbarem $f$ und lokalem Minimum $x^*$ gilt $\nabla f(x^*) = 0$. Solche Punkte heißen stationär.

Hinreichend 2. Ordnung

Gilt $\nabla f(x^*) = 0$ und $\nabla^2 f(x^*) \succ 0$, so ist $x^*$ ein striktes lokales Minimum.

Bei konvexem $f$

$\nabla f(x^*) = 0$ ⟹ $x^*$ ist globales Minimum (notwendige Bedingung wird hinreichend).

2 · Lineare Optimierung (LP)

2.1 Standardform

$$ \min c^\top x \quad \text{u.d.N.} \quad A x = b,\; x \ge 0 $$

Jedes LP kann in Standardform gebracht werden:

  • $\max c^\top x \to \min -c^\top x$.
  • $a^\top x \le \beta \to a^\top x + s = \beta,\; s \ge 0$ (Schlupfvariable).
  • $a^\top x \ge \beta \to a^\top x - s = \beta,\; s \ge 0$ (Überschussvariable).
  • Freie Variable $x$: schreibe $x = x^+ - x^-$ mit $x^\pm \ge 0$.

2.2 Geometrie

$\mathcal{F} = \{x \ge 0 : Ax = b\}$ ist ein Polyeder (Schnitt von Halbräumen).

Fundamentalsatz der LP

Wenn ein LP eine endliche Optimallösung besitzt, dann existiert eine optimale Ecke (Basislösung) des Polyeders. Beweisidee: Höhenlinien der linearen Zielfunktion sind Hyperebenen — sie berühren ein konvexes Polyeder maximal/minimal in einer Ecke.

2.3 Simplex-Verfahren

Idee: starte in einer zulässigen Ecke, wandere zu benachbarten Ecken mit besserem (kleinerem) Zielwert.

Tableau-Schema

             x_1  x_2 ... x_n  s_1 ... s_m  | RHS
  Basis 1     a    a   ...      0   ...      |  b_1
  Basis 2     a    a   ...      0   ...      |  b_2
  ...
  Z (Kosten)  c    c   ...      0   ...      |  0

Iterationsschritte

  1. Optimalitäts-Test: Sind alle reduzierten Kosten $\bar c_j \ge 0$? → Optimum erreicht.
  2. Pivot-Spalte: wähle $j$ mit $\bar c_j < 0$ (negativster zuerst = klassische Dantzig-Regel).
  3. Pivot-Zeile via Min-Ratio-Test: wähle Zeile $i$ mit minimalem $b_i / a_{ij}$ unter $a_{ij} > 0$. Wenn keine $a_{ij} > 0$: das LP ist unbeschränkt.
  4. Pivotelement: $a_{ij}$. Zeile $i$ teilen, andere Zeilen reduzieren.
Degeneration & Bland-Regel

Wenn beim Min-Ratio mehrere Zeilen denselben Wert haben, kann es zu Zyklen kommen. Bland-Regel: stets kleinster Index bei Tie-Break (Eintritt und Austritt) — garantiert Terminierung.

Komplexität

Worst case exponentiell (Klee-Minty-Würfel), aber in der Praxis hervorragend ($O(m)$ Iterationen typisch). Inneres-Punkte-Verfahren laufen polynomial — werden bei sehr großen LPs eingesetzt.

2.4 Dualität

Definition

Zu jedem LP existiert ein duales LP. Bei primalem $$ \min c^\top x,\; A x \ge b,\; x \ge 0 $$ ist das Dualproblem $$ \max b^\top y,\; A^\top y \le c,\; y \ge 0. $$

Konstruktionsregeln (primal → dual)

Dualitätssätze
  • Schwache Dualität: Für jede primal-zulässige $x$ und dual-zulässige $y$: $b^\top y \le c^\top x$.
  • Starke Dualität: Hat das primale Problem ein endliches Optimum, so auch das Duale, und die Werte sind gleich.
  • Komplementärer Schlupf: Bei Optimum gelten $x_j^* (c_j - A_{\cdot j}^\top y^*) = 0$ und $y_i^* (A_{i \cdot} x^* - b_i) = 0$ — d.h. straffe NB ↔ aktive Dualvariable.
Vollständiges Beispiel

Primal: $\max 3x_1 + 2x_2$ u.d.N. $x_1 + x_2 \le 4,\; x_1 + 3 x_2 \le 6,\; x_1, x_2 \ge 0$.

Grafisch: Ecken $(0,0), (4,0), (3,1), (0,2)$. Zielwerte $0, 12, 11, 4$ → Maximum bei $(4,0)$ mit Wert $12$.

Dual: $\min 4 y_1 + 6 y_2$ u.d.N. $y_1 + y_2 \ge 3,\; y_1 + 3 y_2 \ge 2,\; y \ge 0$. Optimum: $y_1^* = 3, y_2^* = 0$ mit Wert $12$. ✓

Komplementärer Schlupf prüfen (alle vier Bedingungen):

  • $x_1^* = 4 > 0$ ⟹ Dual-NB 1 straff: $y_1 + y_2 = 3 + 0 = 3$ ✓ (=$c_1 = 3$).
  • $x_2^* = 0$, also darf Dual-NB 2 locker sein: $y_1 + 3 y_2 = 3 + 0 = 3 \ge 2$ ✓ (Schlupf $1$, kein Widerspruch).
  • Primal-NB 1 straff ($x_1 + x_2 = 4 + 0 = 4$) ⟹ $y_1^* > 0$ erlaubt — und $y_1^* = 3 > 0$ ✓.
  • Primal-NB 2 locker ($x_1 + 3 x_2 = 4 \le 6$, Schlupf $2$) ⟹ $y_2^* = 0$ erzwungen ✓.

3 · Nichtlineare Optimierung

3.1 Eindimensionale Verfahren

  • Bisektion auf $f'$: verlange $f'(a) \cdot f'(b) < 0$, halbiere Intervall. Lineare Konvergenz.
  • Newton auf $f'$: $x_{k+1} = x_k - f'(x_k)/f''(x_k)$. Quadratische Konvergenz lokal; divergiert bei schlechtem Start oder $f''$-Vorzeichenproblem.
  • Goldener-Schnitt: Intervall $[a, b]$, setze $x_1 = a + (1-\rho)(b-a)$, $x_2 = a + \rho(b-a)$ mit $\rho = (\sqrt 5 - 1)/2 \approx 0.618$. Falls $f(x_1) < f(x_2)$, neues Intervall $[a, x_2]$; sonst $[x_1, b]$. Robust, braucht kein $f'$.

3.2 Gradientenabstieg

$$ x_{k+1} = x_k - \alpha_k \nabla f(x_k) $$

Steigt entgegen der Richtung des steilsten Anstiegs. Schrittweite $\alpha_k$ via:

  • Exakte Linesearch: $\alpha_k = \arg\min_\alpha f(x_k - \alpha \nabla f(x_k))$. Teuer.
  • Armijo-Bedingung: $f(x_k - \alpha \nabla f) \le f(x_k) - c_1 \alpha \|\nabla f\|^2$ mit $c_1 \in (0, 1/2)$. Praktisch: starte mit $\alpha = 1$, halbiere, bis Bedingung erfüllt.
  • Wolfe-Bedingung: Armijo + Krümmungsbedingung — Standard in BFGS-Implementierungen.
Konditionszahl-Probleme

Bei stark gestrecktem Niveau-Landscape ($\kappa(\nabla^2 f) \gg 1$) zickzackt der GD entlang der schmalen Richtung. Konvergenzrate $((\kappa - 1)/(\kappa + 1))^2$ — bei $\kappa = 100$ langsam.

Abhilfe: Preconditioning, Momentum (Nesterov), Newton.

3.3 Newton-Verfahren & Quasi-Newton

Newton:

$$ x_{k+1} = x_k - \bigl[\nabla^2 f(x_k)\bigr]^{-1} \nabla f(x_k). $$

Quadratische Konvergenz lokal, schnellster Standard-Solver.

Probleme:

  • Hesse muss positiv definit sein, sonst entsteht keine Abstiegsrichtung.
  • Hesse-Berechnung + Linearsystem $O(n^3)$ pro Schritt.

Modifikationen:

  • Gedämpftes Newton: mit Linesearch $\alpha_k$ stattvoll Schritt $1$.
  • Levenberg-Marquardt: $(\nabla^2 f + \lambda I)^{-1}$ — Hybride aus Newton und Gradientenabstieg.
  • Quasi-Newton (BFGS): approximiert $H^{-1}$ iterativ aus Gradientendifferenzen ohne zweite Ableitungen. Konvergiert superlinear, Standard für mittelgroße glatte Probleme.

3.4 Beschränkte Optimierung: Lagrange & KKT

Lagrange-Funktion $$ L(x, \lambda, \mu) = f(x) + \sum_{i=1}^m \lambda_i g_i(x) + \sum_{j=1}^p \mu_j h_j(x), $$

mit $\lambda_i \ge 0$ (Multiplikatoren der Ungleichungs-NB) und $\mu_j$ beliebig (Gleichungs-NB).

KKT-Bedingungen

Bei regulärem Punkt $x^*$ (z.B. wenn die Gradienten der aktiven NB linear unabhängig sind) gelten folgende Bedingungen notwendig:

  1. Stationarität: $\nabla_x L(x^*, \lambda^*, \mu^*) = 0$ — d.h. $\nabla f(x^*) + \sum \lambda_i^* \nabla g_i(x^*) + \sum \mu_j^* \nabla h_j(x^*) = 0$.
  2. Primal-Zulässigkeit: $g_i(x^*) \le 0,\; h_j(x^*) = 0$.
  3. Dual-Zulässigkeit: $\lambda_i^* \ge 0$.
  4. Komplementärer Schlupf: $\lambda_i^* \cdot g_i(x^*) = 0$ — entweder NB straff ($g_i = 0$) oder Multiplikator $0$.

Bei konvexen Problemen sind KKT-Bedingungen auch hinreichend.

Lesart Komplementärer Schlupf

Eine inaktive NB (straff = false, $g_i < 0$) liefert "freien Raum" — der Multiplikator ist null, die NB ist nicht binding. Eine aktive NB ($g_i = 0$) kann einen positiven Multiplikator haben; dieser misst, wie viel das Optimum sich verbessert, wenn die NB gelockert wird.

Vollständiges KKT-Beispiel

$\min x^2 + y^2$ u.d.N. $x + y \ge 1$, $x, y \ge 0$.

Umform: $g_1 = 1 - x - y \le 0$, $g_2 = -x \le 0$, $g_3 = -y \le 0$.

$L = x^2 + y^2 + \lambda_1 (1 - x - y) + \lambda_2 (-x) + \lambda_3 (-y)$.

Stationarität:

  • $\partial_x L = 2x - \lambda_1 - \lambda_2 = 0$
  • $\partial_y L = 2y - \lambda_1 - \lambda_3 = 0$

Fallunterscheidung: angenommen, das Minimum ist im inneren Quadranten ($x, y > 0$), dann $\lambda_2 = \lambda_3 = 0$. Aus $2x = \lambda_1 = 2y$ folgt $x = y$. Aktive NB $g_1$: $x + y = 1 \Rightarrow x = y = 1/2$, $\lambda_1 = 1$.

Check: $f^* = 1/4 + 1/4 = 1/2$. Komplementärer Schlupf: $\lambda_2 \cdot (-1/2) = 0$ ✓, $\lambda_3 \cdot (-1/2) = 0$ ✓, $\lambda_1 \cdot 0 = 0$ ✓.

3.5 Strafverfahren

Strategie: Nebenbedingungen werden in die Zielfunktion als Bestrafungsterm aufgenommen, dann unbeschränktes Problem lösen.

Quadratische Penalty

$$ P(x, \rho) = f(x) + \rho \sum_j h_j(x)^2 + \rho \sum_i \max(0, g_i(x))^2 $$

$\rho \to \infty$, jedes mal äußere Iteration starten mit vorigem Optimum. Konditionierung wird schlechter mit wachsendem $\rho$.

Barriere (Inneres-Punkte-Verfahren)

$$ B(x, \mu) = f(x) - \mu \sum_i \log(-g_i(x)) $$

$\mu \to 0$. Trajektorie verläuft innerhalb der zulässigen Menge. Standardansatz bei großen LPs und konvexen Problemen.

4 · Gradientenfreie Verfahren (OV-05)

Verwendet nur Funktionswerte. Wann nötig:

  • $f$ nicht differenzierbar oder Gradient teuer/instabil.
  • $f$ verrauscht (Simulationen).
  • $f$ Black-Box (z.B. ML-Hyperparameter).

4.1 Nelder-Mead (Downhill-Simplex)

Verwalte $n+1$ Punkte = Simplex. Sortiere nach $f$-Werten: bester, $\dots$, schlechtester. Operationen:

  • Spiegelung: spiegele schlechtesten am Schwerpunkt der anderen.
  • Expansion: wenn die Spiegelung besser als alle ist, geh noch weiter.
  • Kontraktion: wenn die Spiegelung schlecht, ziehe in Richtung Zentrum.
  • Schrumpfung: wenn alles fehlschlägt, schrumpfe Simplex zum besten Punkt.

Robust, aber langsam und nicht garantiert konvergent (auch für glatte Funktionen). Praktisch gut bis $n \approx 10$.

4.2 Koordinatenabstieg

Optimiere zyklisch einen Parameter, andere fest. Einfach, aber langsam und schlecht bei nicht-separablem $f$.

4.3 Hooke-Jeeves / Pattern Search

Probiere kleine Schritte in Achsenrichtungen. Bei Verbesserung: vergrößere Schrittweite + Pattern Move in die "gefundene" Richtung. Sonst halbiere.

Führe Nelder-Mead per Hand auf $f(x,y) = (x-2)^2 + (y-1)^2$ aus, Start-Simplex $(0,0), (1,0), (0,1)$. Wir machen 4 Iterationen. Ich entscheide jeweils, welche Operation (Spiegelung/Expansion/Kontraktion/Schrumpfung) als nächstes nötig ist. Du korrigierst Geometrie und Reihenfolge.

5 · Metaheuristiken (OV-06)

Stochastische, globale Suchverfahren — keine Optimalitätsgarantie, aber gut bei nicht-konvexen und kombinatorischen Problemen.

5.1 Simulated Annealing

Idee: akzeptiere auch Verschlechterungen mit fallender Wahrscheinlichkeit → entkomme lokalen Optima.

SA(f, x₀, T₀, T_min, α):
  x = x₀; best = x; T = T₀
  while T > T_min:
    for i = 1..L:                        # Innere Schleife
      x' = random_neighbor(x)
      Δ  = f(x') - f(x)
      if Δ < 0 or random() < exp(-Δ/T):  # Metropolis-Kriterium
        x = x'
        if f(x) < f(best): best = x
    T = α * T                            # 0 < α < 1, z.B. 0.95
  return best

Theorie: bei logarithmisch langsamem Cooling ($T_k = c / \log(k+1)$) konvergiert SA mit Wahrscheinlichkeit $1$ zum globalen Optimum — praktisch viel zu langsam. Geometrisches Cooling ($T_{k+1} = \alpha T_k$, $\alpha \approx 0.9$–$0.99$) ist die Praxisvariante.

5.2 Genetische Algorithmen / Evolutionsstrategien

Population $P$ von $\mu$ Individuen. Jede Generation:

  1. Selektion: wähle Eltern. Methoden: Fitness-proportional (Roulette), Turnier, Rang.
  2. Crossover: kombiniere 2 Eltern → 1–2 Kinder. One-Point, Uniform, BLX-α (kontinuierlich).
  3. Mutation: kleine zufällige Änderung. Gauss-Rauschen für reelle Werte, Bit-Flip für Binärrepräsentation.
  4. Ersetzung: $(\mu, \lambda)$: Eltern werden komplett ersetzt; $(\mu + \lambda)$: Eltern + Kinder konkurrieren. Elitismus: das beste Individuum überlebt immer.

Tuning: Populationsgröße, Crossover-Rate, Mutationsrate sind die kritischen Parameter.

CMA-ES = Covariance-Matrix-Adaptation-ES — Stand der Technik für reellwertige Blackbox-Optimierung. Lernt Kovarianzmatrix der Mutationsverteilung adaptiv.

5.3 Particle Swarm Optimization (PSO)

Schwarm von Partikeln $(x_i, v_i)$. Update:

$$ v_i \leftarrow w \cdot v_i + c_1 r_1 (p_i - x_i) + c_2 r_2 (g - x_i) $$ $$ x_i \leftarrow x_i + v_i $$
  • $p_i$ = persönliches Best des $i$-ten Partikels.
  • $g$ = globales Best des Schwarms.
  • $w$ = Inertia (Trägheit), $c_1$ kognitiv (zum persönlichen Best), $c_2$ sozial (zum globalen Best).
  • $r_1, r_2 \sim \text{Uniform}[0, 1]$.

Typische Parameter: $w \in [0.4, 0.9]$ (oft linear fallend), $c_1 = c_2 \approx 2$.

5.4 Ameisenalgorithmus (Ant Colony Optimization)

Für kombinatorische Probleme wie TSP. Pheromonspur $\tau_{ij}$ wird auf guten Lösungswegen verstärkt, verdunstet sonst. Wahl-Wahrscheinlichkeit:

$$ p_{ij} \propto \tau_{ij}^\alpha \cdot \eta_{ij}^\beta $$

mit $\eta_{ij}$ = Heuristik (z.B. $1/d_{ij}$), $\alpha, \beta$ Gewichtungen.

6 · Mehrzieloptimierung (OV-07)

Problem $$ \min_{x \in \mathcal{F}} \bigl(f_1(x), f_2(x), \dots, f_k(x)\bigr) $$

Es gibt keine totale Ordnung — Lösungen werden partiell geordnet via Dominanz.

6.1 Pareto-Dominanz

$x$ dominiert $y$ ($x \prec y$): $$ \forall i: f_i(x) \le f_i(y) \quad \wedge \quad \exists j: f_j(x) < f_j(y) $$

  • Pareto-optimal: nicht dominiert.
  • Pareto-Menge: alle Pareto-optimalen $x$.
  • Pareto-Front: deren Bild $(f_1, \dots, f_k)$ im Zielraum.

6.2 Methoden

Gewichtete Summe (Skalarisierung)

$$ \min \sum_{i=1}^k w_i f_i(x), \quad w_i \ge 0,\; \sum w_i = 1 $$

Findet eine Pareto-Lösung pro Gewichtskombination — durch Variation von $w$ approximiert man die Front.

Limitierung bei nicht-konvexen Fronten

Bei nicht-konvexen Pareto-Fronten kann die gewichtete Summe bestimmte Pareto-Punkte gar nicht erreichen, egal welche Gewichte. Die zugehörige Tangentialebene berührt die Front nur an "ausstehenden" konvexen Teilen.

Abhilfe: $\epsilon$-Constraint oder Multi-Objektiv-EA.

$\epsilon$-Constraint

$$ \min f_1(x) \quad \text{u.d.N.} \quad f_j(x) \le \epsilon_j,\; j \ne 1 $$

Eines der Ziele wird minimiert, alle anderen werden zu Konstraints. Variiert man $\epsilon_j$, durchläuft man die ganze Pareto-Front — auch nicht-konvexe Teile.

Multi-Objektiv Evolutionsalgorithmen

  • NSGA-II (non-dominated sorting GA-II): sortiere Population in Pareto-Fronten. Innerhalb einer Front: Crowding-Distance für Diversität.
  • SPEA2: ähnlich, mit Stärke-basierter Selektion.
  • MOEA/D: zerlege Mehrzielproblem in skalare Subprobleme.

6.3 Qualitätsindikatoren

  • Hypervolumen (HV): Volumen, das die approximierte Front gegen einen Referenzpunkt dominiert. Höher = besser. Einziger Indikator, der streng Pareto-konform ist.
  • IGD (Inverted Generational Distance): mittlerer Abstand wahrer Pareto-Punkte zur approximierten Front. Niedriger = besser. Braucht wahre Front als Referenz.
  • Spread: Maß für Verteilung der Lösungen.

7.4 Schritt 4 — Konvexität entscheidet "lokal vs. global"

Konvex?
  • $f$ konvex auf konvexer $\mathcal F$: Lokale Verfahren reichen (Newton, BFGS). Jedes gefundene lokale Opt. ist global.
  • $f$ konvex, $\mathcal F$ nicht konvex (z.B. ganzzahlige NB): globale Optimierung weiterhin nötig (Branch & Bound).
  • $f$ nicht konvex: Multi-Start mit Newton/BFGS, oder Metaheuristik. Garantie nur stochastisch.

7.7 Zusammenfassende Entscheidungs-Checkliste

  1. Ist alles linear? → LP/Simplex. Fertig.
  2. Diskret? → Branch & Bound (klein), Heuristik (groß).
  3. Konvex?
    • Ableitungen verfügbar → Newton/BFGS.
    • Nur Funktionswerte → Nelder-Mead.
  4. Nicht-konvex?
    • Glatt + lokale Suche reicht? → Multi-Start Newton/BFGS.
    • Globales Optimum wichtig? → SA / GA / PSO / CMA-ES.
  5. Restriktionen?
    • Nur Box → Projektions-Gradient.
    • Gleichungs-NB → Lagrange.
    • Ungleichungs-NB → KKT mit SQP / Inneres-Punkte.
  6. Mehrere Ziele? → Gewichtete Summe (konvex) oder $\epsilon$-Constraint (sonst).

7.8 Häufige Klausur-Falle: "Welches Verfahren?"

Vorgehen in der Klausur
  1. Linearität prüfen: kommen nur lineare Terme vor? → LP.
  2. NB-Typ identifizieren: $\le, \ge, =$, Vorzeichen-Constraints.
  3. Anzahl Variablen + Differenzierbarkeit nennen.
  4. Argument schreiben, warum dieses Verfahren passt (Differenzierbarkeit, Konvexität, NB-Art).
  5. Erst dann rechnen.

8 · Cheatsheet

STRUKTUR
  min f(x)  u.d.N.  g(x) ≤ 0,  h(x) = 0
  konvex: f konvex auf konv. F  ⇒  jedes lokale = global
  Hesse ⪰ 0 ⇔ konvex (für C² f)

LP STANDARD
  min c'x,  Ax = b,  x ≥ 0
  Optimum in Ecke (Fundamentalsatz)
  Simplex: Pivotspalte über reduzierte Kosten < 0
           Pivotzeile via Min-Ratio b_i / a_ij  (a_ij > 0)
           Bland-Regel gegen Zyklen

DUALITÄT
  primal min ↔ dual max
  Primal-NB Σ ↔ Dual-Variable y_i ≥ 0
  Primal-Var x_j ≥ 0 ↔ Dual-NB ≤
  starkes D.: Werte gleich am Optimum
  komplem. Schlupf: x_j (c - A^T y)_j = 0,  y_i (Ax - b)_i = 0

NLP UNBESCHRÄNKT
  notw.:    ∇f(x*) = 0
  hinr.:    ∇f(x*) = 0  ∧  ∇²f(x*) ≻ 0
  GD:       x ← x - α∇f.   Armijo: f(x-α∇f) ≤ f - c₁ α ||∇f||²
  Newton:   x ← x - H⁻¹ ∇f.  quadr. lokal
  BFGS:     H⁻¹ aus Updates der Differenzen.  superlinear

NLP BESCHRÄNKT (KKT)
  L = f + Σ λ_i g_i + Σ μ_j h_j
  (1) ∇_x L = 0
  (2) g ≤ 0,  h = 0
  (3) λ ≥ 0
  (4) λ_i g_i = 0     (komplementärer Schlupf)
  konvex ⇒ KKT auch hinreichend

METAHEURISTIKEN
  SA:    accept worse with prob exp(-Δ/T).  T_{k+1} = αT_k
  GA:    Selektion → Crossover → Mutation → Ersetzung
         (μ, λ) vs (μ + λ) Schemata
  PSO:   v ← wv + c₁ r₁ (p_i - x) + c₂ r₂ (g - x)
  CMA-ES: adaptive Kovarianzmatrix — state of the art

MEHRZIEL
  Pareto x ≺ y ⇔ ∀i f_i(x) ≤ f_i(y) ∧ ∃j ...<...
  gewichtete Summe: konvexe Front voll erreichbar, nicht-konvex unvollständig
  ε-Constraint: kann beide Fronten
  Qualität: Hypervolumen (höher=besser), IGD (niedriger=besser)
